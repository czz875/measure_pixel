# 全面重构设计文档

**日期**: 2026-06-07
**项目**: 图片测量工具 (d:\PycharmProjects\tools)
**范围**: P0 + P1 + P2 全部 23 个问题

## 1. 背景与目标

代码扫描发现 23 个问题：6 个 P0（运行错误/行为错误）、8 个 P1（明显异味）、9 个 P2（架构/可维护性）。其中：
- `app.py` 单文件 1314 行
- `Config` 类被导入但从未实例化
- `StyleManager` 默认样式字段与 `ui_styles.json` 不一致
- 撤销/恢复会丢多边形
- 多图导出图名错位
- 缩放输入框是死代码
- 旧版字段 `point_on_circle` 兼容性缺失

目标：消除所有 P0/P1/P2 问题，统一配置入口，把单文件拆为职责清晰的 6 个模块，加 6 个测试文件覆盖关键回归点。

## 2. 架构重构

### 2.1 模块拆分

| 模块 | 职责 | 来源 |
|---|---|---|
| `src/core/app.py` | 主类 ImageMeasureApp，UI 装配 + 事件分发 | 旧 app.py 协调部分 |
| `src/core/drawing.py` | 7 类标注的 PIL 绘制 | 旧 redraw() 中所有 draw 块 |
| `src/core/picking.py` | 拾取/拖拽点 | 旧 pick_* / on_drag |
| `src/core/tools.py` | 6 种测量模式切换与点添加 | 旧 enable_*_mode / add_*_point |
| `src/core/history.py` | 撤销/恢复栈（含多边形、上限） | 旧 save_state / restore_state / undo / redo |
| `src/core/commands.py` | 文件 I/O / 导出 / 项目加载 | 旧 export_lengths / batch_export / save_project / load_project |
| `src/core/font_loader.py` | 字体加载（含 fallback） | 旧 7 处 try/except 块 |
| `src/core/serializable.py` | to_dict/from_dict 混入基类 | 旧 6 个 dataclass 重复样板 |
| `src/core/constants.py` | 颜色/缩放/单位等魔法值 | 旧硬编码字符串 |

## 3. 配置统一

### 3.1 Config 改造

- `BASE_DIR` 改为指向项目根（`os.path.dirname(os.path.dirname(os.path.dirname(__file__)))`）
- 新增 `UNITS = ('px', 'mm', 'cm', 'in')` 作为单一来源
- `MEASUREMENT.supported_units` 删除（与 `UNITS` 重复）
- `Config` 提供 `get_units()` 类方法供 app.py 调用

### 3.2 StyleManager 修复

`_get_default_styles` 字段名与 `ui_styles.json` 对齐：
- `tooltip_background` / `tooltip_text`（不是 `tooltip_bg` / `tooltip_fg`）
- `toolbar`（不是 `toolbar_bg` / `status_bar_bg`）
- 增加 `toolbar_shadow` 字段
- 默认色值采用与 json 一致的 Office 风格
